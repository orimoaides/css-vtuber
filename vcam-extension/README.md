# CSS VTuber Camera — ブラウザ内仮想カメラ拡張

OBSなどのソフトを**インストールせずに**、CSS VTuberモデルを
Google Meet / YouTube Live(ウェブカメラ配信)の「カメラ」として使うためのChrome拡張。

仕組み: Meet/YouTubeページの `getUserMedia` を差し替えて
「**CSS VTuber Camera**」という仮想カメラデバイスを追加し、
モデルのビューアタブの映像を WebRTC でタブ間転送する。
すべてブラウザ内で完結し、OSへのインストールは不要。

## セットアップ (初回のみ)

1. Chromeで `chrome://extensions` を開く
2. 右上の「**デベロッパーモード**」をON
3. 「**パッケージ化されていない拡張機能を読み込む**」→ この `vcam-extension` フォルダを選択

## 使い方

1. **`start-vtuber.bat` をダブルクリック** (サーバー起動+コントローラーが自動で開く。
   黒い窓は閉じると止まるので開いたまま。手動なら `node serve.js` でも同じ)
2. コントローラーでカメラトラッキング等を開始
3. 「モデルを別窓で開く」でビューア窓を出す
4. ビューア窓の左下 **「🎥 仮想カメラ開始」** をクリック →
   共有ダイアログで**このタブ**を選ぶ (🔴配信中バッジが出る)
5. - **Google Meet**: 設定 → 映像 → カメラ → **CSS VTuber Camera** を選択
   - **YouTube Live**: studio.youtube.com → 作成 → ライブ配信開始 →
     「ウェブカメラ」→ カメラ選択で **CSS VTuber Camera**
6. コントローラー窓での操作(表情・ポーズ・トラッキング)がそのまま配信に反映される

## 注意

- ビューア窓とコントローラー窓は**閉じない**こと (閉じると映像が止まる)
- ビューアの背景色は `controller.html?viewer&bg=00ff00` のようにURLで指定可能
- 「CSS VTuber Camera」を選ぶと、未接続の間は**案内画面**(「ビューア窓の🎥を押してください」)が
  カメラ映像として表示される。4秒ごとに自動再接続し、ビューアが配信を始めると
  **自動でモデル映像に切り替わる**(実カメラへのフォールバックはしない)
- 拡張がサイトで動かないときは、拡張アイコン →「サイトデータの読み取りと変更」→
  「すべてのサイトで」を許可し、タブを再読み込みする(ツールバーの「対応が必要」も承認)
- 会社の管理ポリシーでデベロッパーモードが禁止されている場合は使えない。
  その場合は Meet の「画面共有 → ウィンドウ」でビューア窓を共有するのが代替
- ポートを変える場合は `serve.js` と `manifest.json` の `localhost:8321` を揃えて変更

## 重い時のチェックリスト (Meetでカクつく/映らない時)

配信用途(`?obs`/`?live`/ビューア窓)は **既定で「配信軽量モード」** が効く。
それでも重い時は上から順に確認する。

1. **軽量モードが効いているか**: 配信で映すモデルのURLは `?obs`(controllerの埋め込み既定)
   か `?live`。この2つは自動で軽量プロファイル(縁取りフィルタを外周+目の2枚に統合・
   微小アイドルアニメ停止)になる。見た目は元とほぼ同じで、浮遊/まばたき/口パクは残る。
2. **`?full` を付けていないか**: URLに `?full` を足すと全部入り(重い)に戻る。
   軽量モードを解除したい時だけ使う。普段は付けない。
3. **手の検出(ポーズ)は既定OFF**: Pose検出は重いので既定で切ってある。
   手を映したい時だけ controller の「カメラで手も検出」をONにする(OFFに戻すと軽くなる)。
4. **キャプチャ解像度**: ビューアの `🎥仮想カメラ開始` は **1280x720 / 24fps 上限** に制約済み。
   Retina(2倍)の巨大フレームで詰まる事故を防ぐ。配信バッジに **実測fps** が小さく出るので、
   極端に低い(数fps)なら下の項目を確認。
5. **他アプリでカメラ/GPUを食っていないか**: Zoom/OBS/Teams等を閉じる。
   ブラウザのタブを減らす。ノートPCは電源接続+高パフォーマンス。
6. **接続が「待機中」から進まない**: ビューア窓の `🎥` を押したか確認。
   会議側は最大15秒待って再接続する(重い環境向けに延長済み)。数十秒動かない時は
   ビューア窓を再読み込みして `🎥` を押し直す。

> 軽量モードでの実測(弱いマシン相当=ソフトウェア描画+CPU6倍負荷で計測): 縁取りフィルタの
> 再ラスタライズ時間が全部入り比 **約1/10(≈89%減)**、モデル描画コスト合計 **約72%減**。
> 見た目の印象(縁取り・目の縁・まばたき・浮遊)は維持。
