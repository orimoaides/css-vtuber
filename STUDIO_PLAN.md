# 【凍結】CSS VTuber スタジオ実装計画 — 「OBS的なもの」を自作する

> **2026-07-04 方針転換により凍結。** 本プロジェクトは「VTuberモデルを動かすアプリ」であり、
> 配信画面の合成は対象外になった。一度実装した studio.html は削除済み
> (postMessageブリッジはモデル側に残り、controller.html が利用している)。
> 「モデル側の並行改善」の項は controller.html + モデル本体で実装済み。
> 音声リップシンク(M5の一部)とコメント連携等は、必要になったら controller.html 側に足す。

## ゴール
css-vtuber.html のモデルを載せて配信画面を丸ごと組み立てる、ブラウザ製の配信スタジオ
`studio.html` を作る。思想はモデルと同じ: **画面はDOM+CSSで合成、JSは状態の注入だけ**。

## アーキテクチャ
```
studio.html
├ シーン (fullscreenのDOMツリー、CSSで合成)
│  ├ 背景レイヤー (単色/グラデ/画像/動画/透過)
│  ├ モデルレイヤー (<iframe src="css-vtuber.html?live"> を配置・拡縮)
│  ├ オーバーレイ (テロップ、時計、枠、ロゴ、コメント欄)
│  └ 演出レイヤー (パーティクル、トランジション)
├ コントロールバー (シーン切替、表情ボタン、音量メーター) ※配信画面外 or ホットキー
└ 状態: localStorage に全レイアウト保存 (シーンプリセット)
```
モデルは iframe 埋め込みが第一候補 (モデル本体を無改造で使える)。
表情やパラメータはURLパラメータ + `postMessage` API をモデル側に追加して外から叩く。

## フェーズ計画

### M4: スタジオ基盤 (最初のマイルストーン) — ✅実装済み (2026-07)
実装内容: `studio.html` (背景グラデ+ドット装飾/モデルiframeのドラッグ移動・ホイール拡縮/
シーン3種を localStorage 保存/テロップ・時計・MUTEバッジ/ホットキー 1〜7・S・M・H/
`?live`=パネル非表示・`?scene=名前`)。モデル側に postMessage 受信口を追加済み
(`{type:'exp',name}` / `{type:'var',key,value}`、ロード時に親へ `{type:'vtuber-ready'}` を送信)。
レイヤーの汎用追加/削除/並び替えUIは未実装 (現状は背景/モデル/オーバーレイの固定3層)。
- studio.html 新規作成: シーングラフ(レイヤーの追加/削除/並び替え/位置/拡縮をドラッグ編集)
  — エディタで作った選択枠・ドラッグ・Undoの仕組みをそのまま流用できる
- モデルiframe埋め込み + postMessage ブリッジ (モデル側に受信口を追加:
  `{type:'exp', name:'smile'}` / `{type:'var', key:'--zoom', value:0.9}`)
- シーンプリセット: 「配信中」「待機」「BRB」等をlocalStorageに保存、ワンキー切替
- ホットキー: 1〜7=表情、S=シーン切替、M=ミュート表示

### M5: 演出
- テロップ/ローワーサード (CSSアニメで入退場)
- マイク音量リップシンク補助: AudioContext RMS → --mouth-open にブレンド
  (カメラの口検出と max() 合成。マスク着用時や横向きでも口が動く)
- BGM/SEサウンドボード、CSSパーティクル(紙吹雪・ハート等をコメントやキーでトリガー)
- 待機画面ループアニメ

### M6: 映像出力 (3案併記、上から順に低コスト)
- **案A (推奨・即戦力)**: OBSのブラウザソースで studio.html を取り込む。
  自作部分は「シーン合成UI」まで、エンコード/配信はOBSに任せる
- **案B (録画内蔵)**: `canvas.captureStream` は使えない(DOM合成のため)ので
  `getDisplayMedia` + `MediaRecorder` でスタジオタブをWebM録画。動画投稿用途に十分
- **案C (完全自前配信・上級)**: ローカル中継サーバー (node-media-server + ffmpeg)。
  MediaRecorderのチャンクをWebSocketで送り、ffmpegでRTMPに変換してYouTube/Twitchへ。
  ブラウザ単体ではRTMPを話せないため、ここだけはNode常駐が必要

### M7: コメント連携
- YouTube Live Chat API / Twitch IRC(WebSocket) でコメント取得
- コメント欄オーバーレイ表示 + キーワードで演出トリガー (例:「88888」で紙吹雪)
- コメント読み上げ (SpeechSynthesis) はオプション

### モデル側の並行改善 (随時)
- 呼吸アニメを .bodygrp に再配線 (旧teeは非表示のため現在呼吸なし)
- 髪の遅延揺れ (首の回転に対して髪blobがtransition遅延でついてくる)
- 腕のアイドルスイング、まばたきの二段カーブ
- 表情の追加 (ウィンク、照れ、ドヤ顔 等)

## 推奨着手順
1. M4のpostMessageブリッジ (モデル側20行程度の追加) — これが全ての土台
2. studio.html の最小版: 背景+モデル+ホットキー表情切替
3. OBSブラウザソースで実配信テスト (案A)
4. 以降、M5→M7を配信しながら育てる。案B/Cは必要になってから
